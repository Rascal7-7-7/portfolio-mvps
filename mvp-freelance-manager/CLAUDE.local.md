# Local Rules — mvp-freelance-manager

## 価値メッセージ
「個人事業の業務をまとめて管理できる」

## Product Goal
案件・請求・タスク・収支の統合感をダッシュボードで見せ、「バックオフィスがひとつに集まった」価値を伝える。

## 優先フロー
1. ダッシュボードを開く（案件数・今月売上・未払い請求などが一目でわかる）
2. 案件一覧でステータスを確認する（進行中・完了・失注）
3. 請求書を作成する（案件に紐づけて金額・期日を設定）
4. タスク一覧で今日やることを確認する
5. 収支サマリーで月次推移を確認する

## Priority
- ダッシュボードの統合感（これが単なるToDoアプリとの差）
- 案件・請求・タスクの横断的な見え方
- 月次収支のグラフや集計表示
- ダミーデータで「稼働中の事業感」を出すこと

## Avoid
- 会計ソフト連携・確定申告機能
- 顧客管理の複雑化
- 認証・複数ユーザー対応
- 単なるToDoアプリに見えるUI構成

## コマンド
`/mvp-plan` → `/auto-build-crud` → `/demo-check`

## Execution Rules（最重要）

以下の順番で必ず実行すること：

1. `/mvp-plan` を実行し、Goal / Flow / Screen / Data を確定する
2. `/auto-build-*` でUIまたはCRUDを実装する
3. `/demo-check` でデモ品質を検証する

### 絶対ルール

- 実装前に必ず plan を出すこと
- デモフローが3分以内に収まらない機能は実装しない
- UIは「価値が伝わるか」を最優先する
- DBや認証は後回し（必要になったら追加）

### NG行動

- いきなりコードを書き始める
- 完成前に最適化する
- スコープを勝手に広げる
