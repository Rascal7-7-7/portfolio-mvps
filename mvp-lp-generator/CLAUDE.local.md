# Local Rules — mvp-lp-generator

## 価値メッセージ
「LP制作を爆速で代替できる」

## Product Goal
店舗情報を入力するだけで、デモ品質のランディングページが即座に生成される体験を見せる。

## 優先フロー
1. 入力フォームに店舗情報を入力する（3〜5項目）
2. 「生成する」ボタンを押す
3. ローディング演出が出る
4. プレビュー画面にLPが表示される
5. セクションを編集できる

## Priority
- 入力からプレビュー表示までの速度を最優先
- プレビューUIの見た目クオリティ（これが価値の証明）
- セクション単位の編集感（「使えるもの」感を出す）
- テンプレート方式でまず動かす

## Avoid
- 複雑なCMS的アーキテクチャ
- 保存・公開・管理機能（デモスコープ外）
- 認証・DBの複雑化
- 入力項目が6つ以上になること

## コマンド
`/mvp-plan` → `/auto-build-generator` → `/demo-check`

## Execution Rules（最重要）

以下の順番で必ず実行すること：

1. `/mvp-plan` を実行し、Goal / Flow / Screen / Data を確定する
2. `/auto-build-*` でUIまたはCRUDを実装する
3. `/demo-check` でデモ品質を検証する

### 絶対ルール

- 実装前に必ず plan を出すこと
- デモフローが3分以内に収まらない機能は実装しない
- UIは「価値が伝わるか」を最優先する
- DBや認証は後回し（必要になったら追加）

### NG行動

- いきなりコードを書き始める
- 完成前に最適化する
- スコープを勝手に広げる
