# Local Rules — mvp-estimate-manager

## 価値メッセージ
「営業の手作業を自動化できる」

## Product Goal
見積作成フロー・金額計算・ステータス管理・送付済み感を見せ、「営業業務の時短」を体感させる。

## 優先フロー
1. 見積一覧画面を開く（ステータス・金額が一覧で見える）
2. 新規見積を作成する（顧客名・品目・数量・単価を入力）
3. 合計金額が自動計算される
4. 見積書プレビューを確認する（送付できる感）
5. ステータスを変更する（下書き→送付済み→受注/失注）

## Priority
- 見積作成フォームと自動計算（これが価値の核心）
- 見積書プレビューのリアルな見た目（「送れる」感）
- ステータス管理と一覧の視認性
- 受注率・合計金額などの集計ダッシュボード

## Avoid
- 電子署名・PDF出力（デモスコープ外）
- 顧客管理との連携
- 認証・権限管理
- 複数通貨・税率の複雑な設定

## コマンド
`/mvp-plan` → `/auto-build-crud` → `/demo-check`

## Execution Rules（最重要）

以下の順番で必ず実行すること：

1. `/mvp-plan` を実行し、Goal / Flow / Screen / Data を確定する
2. `/auto-build-*` でUIまたはCRUDを実装する
3. `/demo-check` でデモ品質を検証する

### 絶対ルール

- 実装前に必ず plan を出すこと
- デモフローが3分以内に収まらない機能は実装しない
- UIは「価値が伝わるか」を最優先する
- DBや認証は後回し（必要になったら追加）

### NG行動

- いきなりコードを書き始める
- 完成前に最適化する
- スコープを勝手に広げる
