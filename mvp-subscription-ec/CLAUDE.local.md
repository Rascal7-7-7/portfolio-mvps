# Local Rules — mvp-subscription-ec

## 価値メッセージ
「D2C ECを作れる」

## Product Goal
商品一覧・プラン比較・カート風UI・購入完了風導線を見せ、「サブスクECが動いている」感を伝える。

## 優先フロー
1. 商品一覧を開く（商品・価格・プランが一目でわかる）
2. プラン比較セクションで月額/年額を切り替える
3. プランを選んでカートに追加する
4. カート画面で内容を確認する
5. 「購入する」ボタンで完了画面へ遷移する（決済モック）

## Priority
- 商品一覧・プラン比較の見た目クオリティ（D2C感）
- 月額/年額切り替えなどサブスクらしい表現
- 購入完了画面の「完結した感」
- ダミーデータでリアルな商品感を出すこと

## Avoid
- 実決済処理（Stripe等の本実装）
- 認証・マイページ・注文履歴
- 在庫管理・管理画面
- 複雑なカート状態管理

## コマンド
`/mvp-plan` → `/auto-build-ui` → `/demo-check`

## Execution Rules（最重要）

以下の順番で必ず実行すること：

1. `/mvp-plan` を実行し、Goal / Flow / Screen / Data を確定する
2. `/auto-build-*` でUIまたはCRUDを実装する
3. `/demo-check` でデモ品質を検証する

### 絶対ルール

- 実装前に必ず plan を出すこと
- デモフローが3分以内に収まらない機能は実装しない
- UIは「価値が伝わるか」を最優先する
- DBや認証は後回し（必要になったら追加）

### NG行動

- いきなりコードを書き始める
- 完成前に最適化する
- スコープを勝手に広げる
