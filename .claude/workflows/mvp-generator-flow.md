# MVP Generator Flow

## Goal
入力→生成→プレビューの流れを最短で見せ、「自動でLPが作れる」という価値を伝える。

## Flow

```
Step 1: /mvp-plan
  - MVP Goal を1文で定義する（例: 店舗情報を入力するだけでLPが生成される）
  - User-Visible Core Flow（入力→生成→プレビュー→編集）を整理する
  - 入力フォームの項目を3〜5個に絞る
  - テンプレート方式か AI 生成かを決定する
  - スコープから切るものを明示する（保存・公開・CMS機能など）

Step 2: /auto-build-generator
  - 入力フォーム → 生成処理 → プレビュー画面の順に実装する
  - ローディング表示を必ず入れる
  - 生成結果の編集機能を最低1セクション分は入れる

Step 3: /demo-check
  - フォーム入力からプレビュー表示まで30秒以内か確認する
  - 「入力→即プレビュー」の感動が伝わるかレビューする
  - 改善候補を最大3つ出す
```

## Constraints

- テンプレート方式を優先（AI生成はデモ価値が不足する場合のみ追加）
- 保存・公開機能は今回スコープ外
- 入力フォームは5項目以内
- プレビューはリアルなデザインで表示すること（見た目が命）

## Final Output

- 入力フォーム（3〜5項目）
- 生成プレビュー画面（リアルなLPデザイン）
- セクション単位の編集機能
- ローディング演出
