# MVP CRUD Flow

## Goal
予約・見積・フリーランス管理などの業務システム型MVPを、最短でデモ可能な状態にする。

## Flow

```
Step 1: /mvp-plan
  - MVP Goal を1文で定義する
  - User-Visible Core Flow を番号付きで整理する
  - Minimal Screen List（一覧・詳細・フォームの3画面）を確定する
  - Minimal Data Model（ステータスつきの最小スキーマ）を定義する
  - スコープから切るものを明示する

Step 2: /auto-build-crud
  - データモデル → 一覧画面 → 詳細画面 → 作成フォームの順に実装する
  - ステータス管理とダッシュボード集計を必ず入れる
  - ダミーデータでリアルに見えるようにする

Step 3: /demo-check
  - デモフローを3分以内で辿れるか確認する
  - 「何の業務が、どう楽になるか」が画面から伝わるかレビューする
  - 改善候補を最大3つ出す
```

## Constraints

- 認証不要（デモ用の固定データで可）
- 複雑なバリデーションより画面の完成感を優先
- 一覧→詳細→作成の導線が切れないこと
- ステータス変更がワンクリックで見えること

## Final Output

- 動作する CRUD 画面（一覧・詳細・作成）
- ステータス管理フロー
- ダッシュボード（件数・金額などの集計）
- デモ用ダミーデータ（リアルに見えるもの）
