# MVP EC Flow

## Goal
商品一覧→プラン比較→カート→購入完了の導線を見せ、「D2C ECが作れる」という価値を伝える。

## Flow

```
Step 1: /mvp-plan
  - MVP Goal を1文で定義する（例: サブスク商品を選んで購入完了まで辿れる）
  - User-Visible Core Flow（商品一覧→プラン比較→カート→完了）を整理する
  - Minimal Screen List を確定する
  - 決済はモックUIで済ませることを明示する
  - スコープから切るものを明示する（在庫管理・マイページ・管理画面など）

Step 2: /auto-build-ui
  - 商品一覧・プラン比較・カート・購入完了の順に実装する
  - サブスクらしい「継続プラン」表現を必ず入れる（月額/年額の切り替えなど）
  - 購入完了画面は「完結した感」が出るデザインにする

Step 3: /demo-check
  - 商品選択から完了画面まで3クリック以内か確認する
  - 「ECサイトとして成立している」ことが伝わるかレビューする
  - 改善候補を最大3つ出す
```

## Constraints

- 決済機能は実装しない（「購入する」ボタンで完了画面へ遷移するモックでよい）
- 認証・マイページ・注文履歴は今回スコープ外
- 商品データはダミーJSONで可
- サブスクらしいプラン表現（月額・年額・プラン比較）は必須

## Final Output

- 商品一覧ページ（サムネイル・価格・プラン）
- プラン比較セクション（月額/年額切り替え）
- カートUI（選択内容の確認）
- 購入完了ページ（完結感のあるデザイン）
