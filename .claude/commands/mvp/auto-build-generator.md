# /auto-build-generator

生成系MVP向けの自動構築コマンドです。
対象: LP生成（mvp-lp-generator）

## 重視すること

- input → generate → preview flow（入力して即座に結果が見える）
- editable output（生成結果を編集できる）
- fast demo（起動からプレビューまで30秒以内）
- minimal persistence（DBは後回し。まずUI完結で動かす）

---

## Steps

1. **Goal確認**: 何を生成して何を見せるか1文で再確認する
2. **入力フォーム設計**: 最小限の入力項目を定義する（3〜5項目）
3. **生成ロジック設計**: テンプレート埋め込み or AI生成 のどちらかを選択する
4. **プレビューUI設計**: 生成結果をリアルタイムまたはワンクリックで確認できる画面を作る
5. **編集機能**: 生成後にセクションを編集できるようにする
6. **完成感**: 「完成した」と感じさせるCTAや表示を加える

## Rules

- まずはテンプレート方式を優先し、デモ価値が不足する場合のみAI生成を追加する
- AI生成を採用する場合も、生成部分は差し替え可能な関数またはサービス層に閉じ込める
- プレビューは専用プレビューページまたはプレビューコンポーネントで表示する
- 保存機能はデモ上不要なら省略可（ローカルステートで十分）
- 「生成中...」のローディング表示は必須（デモ映えする）
- 入力直後に最低1回で完成イメージが見えることを優先する

## Output

- 入力フォームの項目定義
- 生成ロジックの方式（テンプレート or AI）
- プレビュー画面の構成
- 編集可能なセクション一覧
- フルコード（省略なし）
- デモ時の説明ポイント3つ
- 次の改善候補（最大3つ）
