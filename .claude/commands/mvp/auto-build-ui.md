# /auto-build-ui

UI重視MVP向けの自動構築コマンドです。
対象: LP生成（mvp-lp-generator）、ECの見せ方部分（mvp-subscription-ec）

## 重視すること

- visible value（開いた瞬間に価値が伝わる）
- screen flow（画面遷移が直感的）
- preview / demo quality（見た目のクオリティ）
- simplified data model（DBは最小限）

---

## Steps

1. **Goal確認**: 何を見せたいか1文で再確認する
2. **画面設計**: デモに必要な画面を列挙し、優先順位をつける
3. **UI構造**: コンポーネント構成を最小単位で設計する
4. **ダミーデータ**: リアルに見えるサンプルデータを定義する
5. **実装**: 見た目から作る（機能より先にUIを完成させる）
6. **プレビュー確認**: 動かして確認できる状態にする

## Rules

- Tailwind CSS でスタイリングする
- shadcn/ui コンポーネントを積極活用する
- レスポンシブは最低限（デスクトップ優先）
- アニメーションは控えめに（デモが重くなるのを防ぐ）
- 状態管理は useState で十分な場合は追加ライブラリ不要

## Output

- 画面一覧とコンポーネント構成
- フルコード（省略なし）
- デモ時の説明ポイント3つ
- 次の改善候補（最大3つ）
