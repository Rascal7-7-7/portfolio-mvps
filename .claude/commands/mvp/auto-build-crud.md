# /auto-build-crud

CRUD重視MVP向けの自動構築コマンドです。
対象: 予約（mvp-booking-system）、見積（mvp-estimate-manager）、フリーランス管理（mvp-freelance-manager）

## 重視すること

- list / detail / form flow（一覧→詳細→作成の導線）
- status management（ステータス変更が見える）
- search / filter（一覧で絞り込める）
- dashboard basics（集計・サマリーが見える）
- minimal schema（テーブル設計は最小限）

---

## Steps

1. **Goal確認**: 何の業務を何のためにシステム化するか1文で再確認する
2. **データモデル設計**: 最小限のテーブル構成を定義する
3. **画面設計**: 一覧・詳細・作成フォームの3画面を基本とする
4. **ステータスフロー**: データがどの状態を持つか定義する
5. **実装**: 一覧画面から作る（最初に全体感を見せられるようにする）
6. **ダッシュボード**: 件数・金額など集計値を1画面に載せる

## Rules

- Server Actions または API Routes でデータ操作する
- SQLite / PostgreSQL / ダミーJSON どれでも可（デモ目的なら配列でも可）
- フォームバリデーションは最低限（空欄チェックのみでも可）
- ページネーションは20件固定でよい
- 削除は論理削除より物理削除でシンプルに

## Output

- データモデル（型定義 or テーブルスキーマ）
- 画面一覧とルーティング
- フルコード（省略なし）
- デモ時の説明ポイント3つ
- 次の改善候補（最大3つ）
