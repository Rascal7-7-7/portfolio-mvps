# /mvp-plan

MVPとして最小で価値を見せるための実装計画を出力します。

## 使い方

このコマンドを呼ぶ前に、以下を1文で伝えてください。
- 対象MVP（例: mvp-booking-system）
- 今回見せたい価値（例: 予約が取れることを見せたい）

---

## 出力フォーマット

### MVP Goal
> このMVPで「誰に」「何を」見せるか、1〜2文で。

### User-Visible Core Flow
取引先がデモを見たときに辿る導線を、番号付きステップで記述する。
例:
1. トップページを開く
2. 予約フォームに入力する
3. 確認画面を見る
4. 完了メッセージが出る

### Minimal Screen / Component List
デモに必要な最小限の画面・コンポーネント一覧。
不要なものは明示的に「今回スコープ外」と書く。

### Minimal Data Model
デモに必要な最小限のデータ構造（型/テーブル）。
ダミーデータで代替できる場合はその旨を記載する。

### Implementation Order
実装する順番を、デモ完成を最優先として並べる。
1. ...
2. ...
3. ...

### What to Cut from Scope
今回のデモには不要なもの一覧。
「あったら良い」ではなく「なくてもデモできる」基準で切る。

---

## 制約リマインダー

- 認証不要（デモ用固定ユーザーで可）
- 決済不要（モックUIで可）
- DB設計より画面フローを優先
- 3分で説明できない機能はスコープ外
