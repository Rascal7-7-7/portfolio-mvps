# Local Rules — mvp-booking-system

## 価値メッセージ
「紙やLINE管理をシステム化できる」

## Product Goal
予約の作成・一覧・ステータス管理・通知風導線を見せ、「業務がシステムに乗った」感を伝える。

## 優先フロー
1. 予約一覧画面を開く（件数・ステータスが一目でわかる）
2. 新規予約を作成する（フォーム入力）
3. 予約詳細を確認する
4. ステータスを変更する（未確認→確認済→完了）
5. 通知風のフィードバックが出る（モック可）

## Priority
- 予約一覧画面のダッシュボード感（管理されている感）
- 予約作成フォームの自然な導線
- ステータス変更がワンクリックで見えること
- 通知はモック演出でよい（実送信不要）

## Avoid
- 実際のメール・LINE送信機能
- カレンダー連携（デモスコープ外）
- 認証・スタッフ管理機能
- 複雑なバリデーション

## Note
- `AGENTS.local.md` に Next.js のエージェントルールあり。Next.js のコード変更前に必ず参照すること。

## コマンド
`/mvp-plan` → `/auto-build-crud` → `/demo-check`

## Execution Rules（最重要）

以下の順番で必ず実行すること：

1. `/mvp-plan` を実行し、Goal / Flow / Screen / Data を確定する
2. `/auto-build-*` でUIまたはCRUDを実装する
3. `/demo-check` でデモ品質を検証する

### 絶対ルール

- 実装前に必ず plan を出すこと
- デモフローが3分以内に収まらない機能は実装しない
- UIは「価値が伝わるか」を最優先する
- DBや認証は後回し（必要になったら追加）

### NG行動

- いきなりコードを書き始める
- 完成前に最適化する
- スコープを勝手に広げる
